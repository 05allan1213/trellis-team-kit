# Backend Development Guidelines

> Best practices for backend development in this project.

---

## Overview

This directory contains guidelines for backend development.

Backend guidelines should document the project's actual backend conventions, not generic best practices. During early bootstrap, keep these files lightweight and fill them only when the project has real backend patterns worth standardizing.

## When to Use This Directory

Use these guidelines only when a task clearly touches backend code, APIs, data access, logging, error handling, or backend quality.

Do not force backend conventions onto a project that does not yet have backend implementation.

---

## How to Fill These Guidelines

For each guideline file:

1. Document your project's **actual conventions** (not ideals)
2. Include **code examples** from your codebase
3. List **forbidden patterns** and why
4. Add **common mistakes** your team has made

The goal is to help AI assistants and new team members understand how YOUR project works.

---

**Language**: All documentation should be written in **English**.

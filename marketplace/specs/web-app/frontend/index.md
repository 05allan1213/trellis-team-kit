This directory contains guidelines for frontend development.

Frontend guidelines should document the project's actual frontend conventions, including technology choices, component patterns, styling approach, state management, hooks, and UI quality expectations.

## When to Use This Directory

Use these guidelines when a task touches frontend UI, components, hooks, state, styling, type safety, or frontend architecture.

For existing frontend projects, follow the existing technology stack, component library, dependencies, file structure, and visual style unless the user explicitly asks for a change.

Avoid broad rewrites when a small, targeted modification satisfies the task.

---

## How to Fill These Guidelines

For each guideline file:

1. Document your project's **actual conventions** (not ideals)
2. Include **code examples** from your codebase
3. List **forbidden patterns** and why
4. Add **common mistakes** your team has made

The goal is to help AI assistants and new team members understand how YOUR project works.

---

**Language**: All documentation should be written in **English**.
